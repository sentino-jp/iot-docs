# Rlink IoT SDK 蓝牙配网技术说明文档

## 1. 概述

Rlink IoT SDK 提供了基于 BLE (Bluetooth Low Energy) 的设备配网功能，支持通过蓝牙为 IoT 设备配置 WiFi 网络信息和云端连接参数。该方案采用双模配网架构（BLE + WiFi），适用于智能玩具、教育机器人等嵌入式设备。

### 1.1 核心特性

- 基于 BLE GATT 协议的数据传输
- 支持大数据分包传输（最大支持任意长度数据）
- JSON 格式的应用层协议
- 支持 WiFi 扫描和配网
- 支持设备绑定和解绑
- 支持 OTA 固件升级
- 配网超时保护机制

### 1.2 技术架构

```
应用层 (rlink_ble_app)
    ↓ JSON 消息处理
传输协议层 (rlink_ble_transfer_protocol)
    ↓ V1 协议分包/解包
GATT 层 (rlink_ble_gatt)
    ↓ BLE 连接管理
硬件抽象层 (Platform Callbacks)
```

## 2. 蓝牙广播协议

### 2.1 广播数据格式

广播数据包含 3 个段：


**段 1: FLAGS**
```
长度: 0x02
类型: 0x01 (FLAGS)
数据: 0x06
```

**段 2: Service UUID**
```
长度: 0x03
类型: 0x02 (UUID16)
数据: 0xA101 (服务 UUID)
```

**段 3: Service Data**
```
长度: 4 + PID 长度
类型: 0x16 (Service Data)
UUID: 0xA101
标志: 0x00
数据: PID 字符串
```

### 2.2 扫描应答数据格式

**段 1: 完整名称**
```
长度: 0x03
类型: 0x09 (完整名称)
数据: "RY"
```

**段 2: 厂商自定义数据**
```
长度: company_len + 1
类型: 0xFF (厂商数据)
数据结构:
  - 厂商 ID: 2 字节 (0x0000)
  - 配网标志: 1 字节 (FLAG)
  - 协议版本: 1 字节 (0x03 = 双模配网)
  - 加密方式: 1 字节 (0x00 = 不加密, 0x01 = 加密)
  - 通信能力: 2 字节 (bit2 = WiFi 2.4G)
  - 标识类型: 1 字节 (0 = UUID, 1 = MAC)
  - UUID/MAC: 最多 19 字节
```

### 2.3 配网标志位 (FLAG) 定义


| 位 | 掩码 | 说明 |
|---|------|------|
| bit7 | 0x80 | 非配网状态标志 (1: 不在配网状态; 0: 在配网状态) |
| bit6 | 0x40 | 绑定标志 (1: 已绑定; 0: 未绑定) |
| bit5 | 0x20 | WiFi 联网状态 (1: 在网; 0: 不在网) |
| bit4 | 0x10 | 通用固件标志 (1: 通用固件; 0: 非通用固件) |
| bit1 | 0x02 | 聚合协议标志 (1: 使用; 0: 不使用) |
| bit0 | 0x01 | 请求连接标志 (1: 请求连接; 0: 未请求连接) |

## 3. BLE 传输协议 (V1)

### 3.1 协议帧格式

每个数据包最大 128 字节，协议头占用 10 字节，有效数据最大 118 字节。

```
+------+------+------+------+------+------+------+------+------+------+------+
| HEAD | TYPE |  SN  |  SN  | TOTAL| TOTAL| LEN  | LEN  |C_LEN | DATA | CRC  |
|  1B  |  1B  |  1B  |  1B  |  1B  |  1B  |  1B  |  1B  |  1B  | N B  |  1B  |
+------+------+------+------+------+------+------+------+------+------+------+
```

**字段说明:**

- **HEAD (1 字节)**: 固定为 0xFF
- **TYPE (1 字节)**: 数据类型，固定为 0x01
- **SN (2 字节)**: 当前包序号（大端序），从 0 开始
- **TOTAL (2 字节)**: 总包数（大端序）
- **LEN (2 字节)**: 有效数据总长度（大端序）
- **C_LEN (1 字节)**: 当前包有效数据长度
- **DATA (N 字节)**: 有效数据
- **CRC (1 字节)**: 校验和（从 TYPE 到 DATA 的累加和）

### 3.2 分包机制


**发送端处理流程:**

1. 计算总包数: `total_pack_num = (len + PER_DATA_LEN - 1) / PER_DATA_LEN`
2. 为每个包添加 10 字节协议头
3. 按 128 字节为单位通过 BLE Notify 发送
4. 每包之间延迟 20ms

**接收端处理流程:**

1. 验证包头 (0xFF)
2. 检查包序号连续性
3. 验证 CRC 校验和
4. 第一包时分配接收缓冲区
5. 拷贝有效数据到缓冲区
6. 最后一包时验证总长度并回调应用层

### 3.3 错误码定义

| 错误码 | 说明 |
|-------|------|
| 1501 | 包头异常 |
| 1502 | 包序号异常 |
| 1503 | CRC 校验失败 |
| 1504 | 设备缓存 1 异常 |
| 1505 | 设备缓存 2 异常 |
| 1506 | 总数据长度异常 |
| 1700 | JSON 解析失败 |
| 1701 | JSON 解析成功，开始联网 |
| 1702 | 服务器连接失败 |
| 1703 | 服务器连接成功 |

## 5. 配网流程详解

### 5.1 配网状态机


```
RLINK_NET_MODE_NONE (待机)
    ↓
RLINK_NET_MODE_BLE_CONFIG (蓝牙配网模式)
    ↓ 收到配网信息
RLINK_NET_MODE_CONFIG_TRY_CONNECT (尝试连接)
    ↓ WiFi 连接成功
RLINK_NET_MODE_STA_START (STA 模式)
```

### 5.2 完整配网流程

**步骤 1: 进入配网模式**

设备端操作：
```c
Rlink_Set_Net_Mode(RLINK_NET_MODE_BLE_CONFIG);
```

执行动作：
- 清除 WiFi 连接标志
- 设置请求连接标志
- 停止当前蓝牙广播
- 更新广播数据（FLAG 标志配网状态）
- 启动蓝牙广播
- 启动 WiFi 扫描
- 设置设备状态为 `DEV_UNPROVISION_STATE`

**步骤 2: APP 扫描并连接设备**

APP 端：
- 扫描 BLE 设备（Service UUID: 0xA101）
- 解析广播数据获取 PID
- 解析扫描应答数据获取设备状态
- 连接设备

设备端回调：
```c
Rlink_Ble_Connect_Envent_Callback(RLINK_BLE_CONNECT_EVENT_CONNECTED, conn_index, dst_addr);
```

**步骤 3: 获取设备信息**

APP 发送：
```json
{
  "type": "device.information.get",
  "ts": 1234567890
}
```

设备响应：
```json
{
  "type": "device.information.get.response",
  "ts": 1234567890,
  "data": {
    "version": "1.0.0",
    "msgType": "poweron",
    "pid": "RTLnG5pi58zlaH55",
    "bind": false,
    "gateway": 0,
    "wifi_mac": "aa:bb:cc:dd:ee:ff",
    "ble_mac": "11:22:33:44:55:66"
  }
}
```

**步骤 4: 获取 WiFi 列表**

APP 发送：
```json
{
  "type": "thing.network.getwifis",
  "scan": 1
}
```

设备响应：
```json
{
  "type": "thing.network.getwifis.response",
  "msgId": "abc123",
  "code": 0,
  "ts": 1234567890,
  "data": {
    "wifis": [
      {
        "ssid": "MyWiFi",
        "rssi": -45,
        "security": true
      }
    ]
  }
}
```

**步骤 5: 发送配网信息**

APP 发送：
```json
{
  "type": "thing.network.set",
  "data": {
    "sid": "MyWiFi",
    "pw": "password123",
    "bid": "bind_id_12345",
    "userId": "user_id_67890",
    "mq": "mqtt.sentino.jp",
    "port": 1883,
    "country": "CN",
    "tz": "Asia/Shanghai",
    "force_bind": 0
  }
}
```

设备处理：
```c
Rlink_Ble_Wifi_Network_Set_Process(data);
```

设备响应：
```json
{
  "type": "thing.network.set.response",
  "code": 0,
  "msgId": "xyz789",
  "ts": 1234567890
}
```

**步骤 6: 连接 WiFi**

设备端自动执行：
```c
Rlink_Wifi_Config_Net_Info(ssid, password);
Rlink_Set_Net_Mode(RLINK_NET_MODE_CONFIG_TRY_CONNECT);
```

执行动作：
- 启动 WiFi STA 连接
- 启动配网超时定时器（60 秒）

**步骤 7: 连接 MQTT**

WiFi 连接成功后：
- 获取 IP 地址
- 停止配网超时定时器
- 连接 MQTT 服务器
- 设置绑定标志
- 保存配网信息到 Flash

### 5.3 配网超时处理

配网超时机制：
- 超时时间：60 秒
- 触发条件：获取到 WiFi 信息但未连接上 MQTT
- 超时动作：
  ```c
  Rlink_Wifi_Sta_Disconnect();
  Rlink_Wifi_Config_Net_Info("", "");
  Rlink_Reset_Config_Net_Info_To_Flash();
  Rlink_System_Reset();
  ```

## 6. 关键数据结构

### 6.1 配网信息结构


```c
typedef struct {
    unsigned char Flag_Wifi_Need_Reconnect;     // WiFi 需要重连标志
    unsigned char Flag_Wifi_Info_Vaild;         // WiFi 信息有效标志
    unsigned char Config_Net_Type;              // 0-WiFi, 1-BLE
    unsigned char Flag_Config_Net_Success;      // 配网成功标志
    unsigned char Flag_Config_Net_Progress;     // 正在配网标志
    unsigned char Flag_Have_Got_Ip;             // 已获取 IP 标志
    unsigned char Flag_Have_Get_Config_Net_Info;// 已获取配网信息标志
    unsigned char Rlink_Config_Net_Flag;         // 蓝牙广播 FLAG
} Wifi_Account_Param_t;
```

### 6.2 BLE 广播信息结构

```c
typedef struct {
    unsigned char Adv_Len;          // 广播数据长度
    unsigned char Scan_Rsp_len;     // 扫描应答数据长度
    unsigned char Adv_Data[32];     // 广播数据
    unsigned char Scan_Rsp_Data[32];// 扫描应答数据
} Rlink_Ble_Adv_Info_t;
```

### 6.3 BLE 分包信息结构

```c
typedef struct {
    unsigned short Total_Sn;    // 总包数
    unsigned short Total_Len;   // 总长度
    unsigned short Recv_Len;    // 已接收长度
    unsigned short Current_Sn;  // 当前包序号
    unsigned short Recv_Sn;     // 已接收包数
    unsigned char Cmd_Type;     // 命令类型
    unsigned char *Buf;         // 数据缓冲区
} Rlink_Ble_Gatt_Packet_Info_t;
```

## 8. 平台适配接口

### 8.1 BLE 回调函数结构

```c
typedef struct {
    void (*Ble_Init)(void);
    int (*Ble_Gatt_Notify_Send)(unsigned char conn_index, 
                                unsigned char *send_data, 
                                unsigned short len);
    int (*Ble_Gatt_Indicate_Send)(unsigned char conn_index, 
                                  unsigned char *send_data, 
                                  unsigned short len);
    int (*Ble_Set_Adv_Data)(unsigned char *adv_data, unsigned char adv_len);
    int (*Ble_Set_Scan_Rsp_Data)(unsigned char *scan_rsp_data, 
                                 unsigned char scan_rsp_len);
    int (*Ble_Adv_Start)(void);
    int (*Ble_Adv_Stop)(void);
    void (*Ble_Get_Local_Addr)(unsigned char *mac_addr);
    int (*Ble_Disconnect)(unsigned char conn_index);
    int (*Ble_Parser_Data)(unsigned char* data, unsigned short len);
} Rlink_Ble_Cbs_t;
```

### 8.2 平台适配步骤

1. 实现 `Rlink_Ble_Cbs_t` 中的所有回调函数
2. 调用 `Rlink_Ble_Cbs_Init()` 注册回调
3. 在 BLE 连接事件中调用 `Rlink_Ble_Connect_Envent_Callback()`
4. 在 BLE 数据接收中调用 `Rlink_Ble_Gatt_Data_In_Callback()`
5. 注册应用层消息处理回调

## 10. 注意事项

### 10.1 安全性考虑

1. **密码传输**: 当前实现未加密，建议在生产环境中使用加密传输
2. **设备认证**: 通过 PID 和 UUID 进行设备识别
3. **绑定保护**: 支持强制绑定标志，防止已绑定设备被重新绑定

### 10.2 性能优化

1. **分包延迟**: 每包之间延迟 20ms，确保 BLE 传输稳定
2. **超时保护**: 配网超时 60 秒，防止设备长时间处于配网状态
3. **内存管理**: 接收大数据时动态分配内存，使用完毕后及时释放

### 10.3 错误处理

1. **CRC 校验**: 每个数据包都进行 CRC 校验
2. **包序号检查**: 确保数据包按顺序接收
3. **超时重试**: WiFi 连接失败后自动重启设备

### 10.4 调试建议

1. 启用日志输出查看配网流程
2. 使用 BLE 抓包工具分析数据传输
3. 检查广播数据和扫描应答数据格式
4. 验证 JSON 消息格式正确性

## 11. 常见问题

### Q1: 设备无法被扫描到？

检查项：
- 确认蓝牙广播已启动
- 检查广播数据格式是否正确
- 确认 Service UUID (0xA101) 是否正确

### Q2: 配网后无法连接 WiFi？

检查项：
- WiFi SSID 和密码是否正确
- 路由器是否支持 2.4G WiFi
- 检查 WiFi 信号强度

### Q3: 数据传输失败？

检查项：
- 确认 MTU 大小（默认 128 字节）
- 检查 CRC 校验是否通过
- 查看包序号是否连续

### Q4: 配网超时？

检查项：
- MQTT 服务器地址和端口是否正确
- 网络连接是否正常
- 检查配网超时时间设置（默认 60 秒）
